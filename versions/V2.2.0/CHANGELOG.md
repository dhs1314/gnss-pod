### RINEX 格式细节

- **格式**：RINEX 2.20，GPS 时间系统
- **观测类型**（9 种）：L1 L2 C1 P1 P2 LA SA S1 S2
- **字段宽度**：16 字符/观测值，末位为 LLI（Loss of Lock Indicator）
- **采样间隔**：10 秒
- **每历元行数**：1 行 epoch header + 2 行/SV
- **L1/L2**：载波相位，**整周数**（~1.15×10⁸ cycles）
- **C1/P1/P2**：伪距，**米**（~2.2×10⁷ m）

### 关键技术发现：C1 vs P1

JPL RL04 处理的相位与 **C/A 码（C1）** 对齐，而非 P 码（P1）。RINEX 同时包含 C1 和 P1 观测值，二者之差为 DCB（差分码偏差），约 0.2–1.0 m。

若使用 P1 作为伪距测量值：
- `B_if = L_if − P_if(P1)` ≈ 0（相位与 P 码差异很小）
- 相位和码测量值几乎等同 → **KF 退化为纯码定位**
- 精度退化至 **4.67 m 3D RMS**（相比 C1 的 1.23 m）

**正确做法**：使用 C1 作为 P1（`P1_use = C1_val if C1_val is not None else P1_val`），使 B_if ≈ −0.7 至 −2.6 m，相位-码分离，KF 可有效约束解算。

## [V2.1.0] — 2026-05-30

### 概述

**Kalman 滤波 PPP**。在 V2.0.0 Two-Pass 架构基础上，将 Pass 2 的逐历元独立 WLS 替换为 Kalman 滤波器，将浮点模糊度 B_if 作为持久状态变量跟踪。支持新升起卫星的在线初始化（B_if + sv_bias），实现全弧段覆盖。

### 重大变更（相对 V2.0.0）

| 项目 | V2.0.0 (Two-Pass WLS) | V2.1.0 (Kalman Filter) |
|------|----------------------|------------------------|
| **状态估计** | 逐历元独立 WLS | Kalman 滤波（预测+更新） |
| **B_if 处理** | 静态常数（前 30 分钟估计） | 持久状态，随机游走跟踪 |
| **新 SV** | 丢弃（无偏差估计） | 在线初始化（缓冲 P_r 5 个历元） |
| **对流层** | 逐历元估计 | 持久状态，随机游走 Q=1e-6 |
| **4h 弧段覆盖** | 40-96% | **100%（481/481 历元）** |

### 算法细节

#### Kalman 滤波状态

```
x = [dX, dY, dZ, clk_r, trop_wet, B_G01, B_G02, ..., B_G32]
     |____ ephemeral ____| |________ persistent _________|
```

- **Ephemeral（4 状态）**：dX, dY, dZ, clk_r — 无先验，每历元独立估计
- **Persistent（1+N 状态）**：trop_wet, B_Gxx — 随机游走，跨历元继承

#### 过程噪声（30s 间隔）

| 状态 | Q (m²/s) | 物理含义 |
|------|---------|---------|
| trop_wet | 1e-6 | ~1cm/√hr 随机游走 |
| B_if | 1e-4 | 跟踪 ~1m/4h 漂移 |

#### 测量模型

```
Phase (σ=0.01m): L_if + clk_sv - rho_corr - sv_bias = -e·dr + clk_r + trop·mf + B_sv
Code  (σ=0.30m): P_if + clk_sv - rho_corr - sv_bias = -e·dr + clk_r + trop·mf
```

- sv_bias 从所有可用历元预估计（初始窗口或在线缓冲）
- B_if 初始化为 `median(L_if - P_if)`，初始不确定性 σ=0.7m

#### SV 生命周期管理

```
新 SV 出现:
  → 缓冲 L_if-P_if (B_if 初始化, 3 epochs)
  → 缓冲 P_r (sv_bias 估计, 5 epochs)
  → 加入 KF 状态

SV 消失 > 10 epochs:
  → 从状态中移除
  → 重新出现时重新初始化
```

## [V2.0.0] — 2026-05-30

### 概述

**Two-Pass Float-Ambiguity PPP**。基于 JPL RL04 GPS1B 数据（相位以米为单位存储，电离层平滑、码对齐），使用 IGS Final SP3 精密星历，通过两次通滤波器估计浮点模糊度和每星偏差，实现 GRACE-FO 约 0.7-2m 3D RMS 的定轨精度。

### 重大变更（相对 V1.0.0）

| 项目 | V1.0.0 | V2.0.0 |
|------|--------|--------|
| **星历** | 广播星历（简化圆轨道） | IGS Final SP3 精密星历 |
| **数据源** | GNV1B + 自定义观测 | JPL RL04 GPS1B（L1/L2 伪距+相位） |
| **插值** | 无（圆轨道解析解） | 10 点 9 阶重心 Lagrange 插值 |
| **PPP 方法** | 单次最小二乘 | Two-Pass 浮点模糊度 + 每星偏差纠正 |
| **状态估计** | [X, Y, Z, clk, trop] | [dX, dY, dZ, clk, trop] |
| **精度 (0.5h)** | ~11.8 cm 3D RMS（广播星历） | **~0.7-1.9m 3D RMS**（实测，码噪声限制） |
| **参考帧** | GNV1B 线性化 | GNV1B 线性化 |

### 关键技术细节

#### 数据格式发现

JPL RL04 GPS1B 的 `L1_phase` / `L2_phase` **以米为单位存储**，而非载波周数。该数据经过电离层平滑并与伪距对齐（`L1_phase ≈ P1`，差异约 0.1m）。这意味着：

- **无法进行整周模糊度解算** — 相位已丧失整数特性
- **精度受码噪声限制** — 约 0.3m 水平
- **浮点模糊度时变** — 平滑算法引入缓慢漂移（0.2-1.7m / 4h）

#### 算法流程

```
Pass 1（前 30 分钟）:
  ├── 计算 B_if = median(L_if - P_if)      ← 浮点模糊度（每 SV）
  ├── 计算 sv_bias = median(P_if + clk - rho) ← 每星 DCB 偏差
  └── 过滤: std(B_if) > 5m 的 SV 被剔除

Pass 2（逐历元 WLS）:
  ├── 相位修正: L_c = L_if + clk_sv - rho - B_if - sv_bias
  ├── 码:         P_c = P_if + clk_sv - rho
  ├── 状态: [dX, dY, dZ, clk_r, trop_wet]
  ├── 相位权重: 1/0.01² = 10000  （压制码噪声）
  ├── 码权重:   1/0.3²  = 11.1
  └── 单次 WLS 求解（线性问题，无需迭代）
```

#### 物理改正

| 改正项 | 方法 |
|--------|------|
| **光行时** | 5 次迭代，发射时刻插值 SP3 |
| **Sagnac 效应** | `(ω_E/C) × (x_s·y_r − y_s·x_r)` |
| **相对论** | SP3 时钟已含周期项；无需额外改正 |
| **对流层湿延迟** | 状态估计，映射函数 `1/sin(el)` |
| **电离层** | 无电离层组合 `α·L1 + β·L2` |

## [V1.0.0] — 2026-04-24

### 概述

首个正式版本。实现 GRACE-FO 卫星精密单点定位（PPP），包含完整的相对论改正、Sagnac 效应和天线 PCV 校正，并通过 2024 年 5 月全月数据完成验证。

---

### 功能与算法

#### 输入
| 字段 | 说明 |
|------|------|
| **GRACE-FO 轨道** | ISDC/GFZ GNV1B 产品（ECEF 坐标，RL04，2024年5月全月） |
| **GPS 广播星历** | 简化圆轨道（21颗GPS卫星，M0/Ω0/ω/e/i/sqrtA） |
| **采样间隔** | 30 秒 |
| **每段时长** | 4 小时 |
| **观测类型** | 伪距 + 载波相位（电离层-free 双频组合） |

#### 核心算法

| 改正项 | 实现 |
|--------|------|
| **光行时改正** | 迭代 2 次，接收端补偿地球自转（旋转矩阵） |
| **Sagnac 效应** | `(Ω_E/C) × (x_sat·y_rcv − y_sat·x_rcv)` |
| **相对论时钟** | Einstein 项 `r·v/C` + 二阶 Doppler `v²/2C` − 13 m 重力红移 |
| **天线 PCV** | GPS 卫星天线相位中心变化（高度角余弦模型） |
| **电离层-free** | `L_IF = α·L1 + β·L2`，`α = F1²/(F1²−F2²)`，`β = −F2²/(F1²−F2²)` |
| **对流层** | Saastamoinen 干延迟 + 随机游走湿延迟（状态估计） |
| **高度角定权** | `w = 1/(σ² / (sin²(el) + 0.01))` |
| **PPP 解算** | 最小二乘，状态 `[X, Y, Z, clock, trop_wet]`，5 未知数 |
| **过滤条件** | 3D 误差 > 1 m 的发散历元被剔除 |

#### 输出

每行一条历元记录：
```
time, dE_m, dN_m, dU_m, d3_m, n_sat
```
- `dE/dN/dU`：ENU 坐标系下与 GNV1B 参考轨道的偏差（米）
- `n_sat`：有效卫星数（高度角 > 10°）

### 已知限制

1. **GPS 卫星轨道**：使用广播星历（简化圆轨道），精度约米级；建议换用 WHU ultra-rapid SP3（~5 cm）提升精度

4. **参考轨道**：以 GNV1B 为"真值"对比，GNV1B 本身有 ~2 cm 误差
